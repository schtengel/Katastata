# Katastata

### Общее описание системы

Katastata представляет собой клиент-серверное приложение для анализа времени использования программ на компьютере.

### Система должна состоять из следующих компонентов:

- Desktop-клиент (WPF) — отвечает за пользовательский интерфейс, отслеживание процессов операционной системы (ОС) и отправку данных на сервер.
- Backend API (ASP.NET Core) — принимает данные от клиента, сохраняет их в базе данных (БД) и предоставляет API для получения статистики.
- База данных (Microsoft SQL Server) — хранит информацию о пользователях, программах, сессиях и настройках.

### Работа системы происходит следующим образом:

1. Клиентское приложение отслеживает запущенные процессы в операционной системе.
2. При запуске программы создается новая Session.
3. При завершении программы Session закрывается.
4. Клиент отправляет данные о сессиях на сервер через API.
5. Сервер сохраняет данные в базе данных.
6. Пользователь может просматривать статистику через интерфейс клиента.

### Цель системы

Программа предназначена для анализа времени, проведенного пользователем за компьютером и в отдельных приложениях, а также для формирования статистики и контроля времени использования программ.

### Целевая аудитория

- пользователи, желающие контролировать время работы за ПК
- разработчики и специалисты, работающие с большим количеством приложений
- пользователи, применяющие методы тайм-менеджмента

### Границы системы

Система отслеживает только **desktop-приложения**, запущенные в операционной системе. Веб-активность внутри браузера не анализируется.

---

### Стек технологий приложения:

Backend: [ASP.NET](http://ASP.NET) Core Web API

Client: WPF (или аналоги с кроссплатформенностью), MVVM, HttpClient

DataBase: Microsoft SQL Server, Entity Framework Core, dbdiagram.io

Documentation: Swagger
